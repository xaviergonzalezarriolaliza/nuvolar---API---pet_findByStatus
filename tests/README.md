# Petstore API Testing Suite

This repository contains automated functional and security tests for the [Swagger Petstore API](https://petstore.swagger.io/), built with **Playwright** and **TypeScript**.

## 🚀 LLM-Enhanced Testing

Recent updates to this repository include **AI-driven test case generation**. We have integrated Large Language Models (LLMs) to identify complex edge cases that go beyond traditional boundary analysis, focusing on security vulnerabilities and unconventional input handling.

### AI-Generated Edge Cases
The test suite in `tests/findPetsByStatus.spec.ts` now includes a data-driven loop utilizing cases generated by Gemini:
- **Security Injection:** SQL injection and XSS payload attempts.
- **Data Integrity:** Null byte injections and path traversal attempts.
- **Encoding:** URL-encoded special characters and emoji support.
- **Stress:** Extremely long string parameters (1000+ characters).

### Prompts Used for Generation
To maintain transparency and allow for future expansion, the following prompts were used to generate the current `aiGeneratedEdgeCases` dataset:

1. **OpenAPI Analysis Prompt:**
   > "Review the OpenAPI schema for the `GET /pet/findByStatus` endpoint. Generate unique edge cases including SQL injection attempts, XSS payloads in query parameters, unexpected character encodings, and logical contradictions. Format the output as a JSON array."

2. **TypeScript Implementation Prompt:**
   > "Generate 10 TypeScript objects for API edge cases including emoji, unicode, and escape characters for the 'status' parameter to be used in a Playwright test loop."

## 🛠 Setup and Execution

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Run tests:**
   ```bash
   npx playwright test
   ```

3. **View reports:**
   ```bash
   npx playwright show-report
   ```
